# Architecture

Top-level layout:

```
NixOS/
├── flake.nix              flake inputs + mkSystem builder
├── flake.lock
├── hosts/                 host-specific (identity + hardware)
├── users/                 user identity + module composition
├── modules/               system-level (NixOS) profile modules
├── home-modules/          per-user (home-manager) modules
└── notes/                 (this vault)
```

## The four trees

### `hosts/<name>/`
Per-host identity. Each host directory contains:
- `configuration.nix` — pure imports list, no settings
- `host.nix` — hostname + per-host typed options ([[Modules/Host options]])
- `hardware.nix` — picks `modules/hardware/*.nix` à la carte + sets host-specific values (PCI bus IDs, etc.)
- `hardware-configuration.nix` — auto-generated by `nixos-generate-config`
- `users.nix` — imports `users/<name>/` for every user that should exist on this host

See [[Hosts/Overview]].

### `users/<name>/`
User identity + which home-modules they import:
- `default.nix` — imports `modules.nix` + lists home-manager imports
- `modules.nix` — `users.users.<name>` settings (groups, shell, linger)
- `dirs.nix` — XDG directory layout via `systemd.tmpfiles`

See [[Users/Overview]].

### `modules/`
System-level (NixOS) modules. Reusable across hosts — no hostname or hardware specifics. Each subdir is a capability; each file inside is one concern.

```
modules/
├── audio/           pipewire
├── boot/            grub-efi
├── core/            nix, locale, security, shell
├── desktop/         wm/, integrations/, tools/, quirks/, fonts, gaming, remote-access
├── hardware/        bluetooth, graphics, nvidia, power-profiles, touchpad (no default.nix — opt-in per host)
├── host/            local.host typed options (displays, …)
└── network/         networkmanager, ssh, localsend
```

Subdirs with `default.nix` import everything inside; hosts can import the whole subdir for convenience or pick individual files for à la carte. See [[Modules/Overview]].

### `home-modules/`
Per-user (home-manager) modules. Capability-grouped files.

```
home-modules/
├── base.nix
├── cli/             fish, git, superfile, neovim, appimage, compression, flatpak
├── desktop/         apps/, services/, tools/, art/, wm/, xdg
├── dev/             scripts, AI tools, nvf wrapper
└── themes/          palette attrsets (catppuccin, dark, …)
```

See [[Home modules/Overview]].

## Convention: `default.nix` is imports-only

Every subdir with a `default.nix` uses it purely to aggregate `imports`. No content lives there. This means:

- Importing a subdir gives the whole capability bundle
- Importing a single file gives just that one concern

The exception is `modules/hardware/` — no `default.nix`, because no host wants every hardware module. Hosts pick individually.

## Convention: hosts/ and users/ contain identity, not packages

Per [[Principles]]:
- `hosts/<name>/` defines who this machine is (hostname, hardware, which modules it wants)
- `users/<name>/` defines who this user is (groups, shell, which home-modules they want)
- Neither installs packages or sets config the way modules do. They COMPOSE modules.
